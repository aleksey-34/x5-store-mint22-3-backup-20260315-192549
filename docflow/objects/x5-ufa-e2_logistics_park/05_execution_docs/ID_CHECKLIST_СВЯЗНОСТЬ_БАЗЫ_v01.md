# Единый чеклист связности базы ИД

Объект: Логистический парк, г. Уфа, Этап 2  
Дата: 20.03.2026  
Цель: контроль полноты цифровой базы (без ориентации на печать)

## 1. Обязательные поля каждой записи

| Поле | Обязательно | Проверка |
|---|---|---|
| Номер документа | Да | Не пусто, уникален в разделе |
| Дата документа | Да | Формат ДД.ММ.ГГГГ |
| Тип документа | Да | АОСР/АООК/Журнал/Акт/ППР/Паспорт |
| Ответственный | Да | ФИО + роль |
| Статус | Да | GREEN/BLUE/YELLOW/NEW |
| Скан/вложение | Да | Файл прикреплен и открывается |
| Связанные документы | Да | Есть минимум 1 ссылка |

## 2. Минимальные связи по типам документов

| Документ | Обязательные связи |
|---|---|
| АОСР монтаж | Акт разбивки осей, Журнал монтажа, Паспорт материала/бетона, ИГС |
| АОСР омоноличивание | Журнал замоноличивания, Паспорт бетона, ИГС |
| АООК | 1+ АОСР, ИГС, протоколы испытаний |
| Журнал входного контроля | Паспорта/сертификаты материалов |
| Акт разбивки осей | Захватка/оси, последующие АОСР |
| Протоколы испытаний | Привязка к АОСР/АООК и дате бетонирования |

## 3. Контроль необработанных документов

Папка контроля: `10_scan_inbox/`

| Статус | Критерий | Действие |
|---|---|---|
| new | файл загружен, не обработан | назначить ответственного |
| parsed | извлечены поля OCR | проверить корректность |
| linked | связка с ИД создана | оставить в базе |
| manual_review | OCR/структура не распознаны | ручная разметка в течение 24 ч |

## 6. Правило приемки в базе

Документ считается принятым только если:

- заполнены номер и дата;
- прикреплен скан/файл;
- проставлен ответственный;
- есть связи с документами цепочки;
- статус не ниже BLUE для согласованных позиций.
