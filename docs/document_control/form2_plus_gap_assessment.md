## 2. Что уже автоматизировано

- Генерация структуры объекта и досье сотрудников.
- Реестр и хранение приказов с экспортом в PDF.
- Прием и архивирование сканов (`AWR`, `PASSPORT`, `ORDER`) с OCR.
- Базовый API для документов, графиков, журнала событий и Telegram-правил.
- Локальная LLM в контуре проекта через API:
  - `GET /local-llm/status`
  - `POST /local-llm/chat`

## 3. Ближайший приоритет закрытия пробелов

1. Сформировать шаблоны и рабочие файлы для ППР/ППРв/акт-допуска.
2. Открыть производственные журналы в `04_journals/production` и ввести стартовые записи.
3. Начать загрузку сканов удостоверений в досье сотрудников (через `scan-to-inbox` + `ingest`).
4. Заполнить `06_normative_base` актуальными НПА (PDF или ссылки + реестр).
5. Запустить ежедневный TODO-контур по чек-листу с отметкой "сделано/не сделано".
