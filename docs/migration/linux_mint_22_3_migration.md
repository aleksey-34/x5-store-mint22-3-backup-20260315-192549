# Переход проекта на Linux Mint 22.3

## Цель
Перенести текущий рабочий проект X5_Store с Windows на Linux Mint 22.3 без потери данных, шаблонов и темпа выпуска документов.

## Что уже подготовлено
- Добавлен Linux-скрипт запуска API: scripts/run_api_clean.sh
- Добавлен Linux-скрипт подготовки окружения: scripts/setup_linux_mint_22_3.sh
- Добавлена VS Code task: Run FastAPI API (Linux Mint)
- Зафиксирована история рабочего чата в docs/chat_history/ для восстановления контекста

## Чеклист перед переносом
1. Проверить, что все изменения закоммичены и запушены в удаленный репозиторий.
2. Проверить наличие папки docflow/objects/x5-ufa-e2_logistics_park в репозитории.
3. Проверить, что в git есть файлы шаблонов из docs/templates/orders.
4. Если есть локальные секреты в .env, перенести их вручную на новое устройство (не коммитить в git).

## Разворачивание на Linux Mint 22.3
1. Установить Git и VS Code.
2. Клонировать репозиторий.
3. Выполнить:

```bash
bash scripts/setup_linux_mint_22_3.sh
```

4. Запустить API:

```bash
bash scripts/run_api_clean.sh --port 8000 --reload
```

5. Проверить открытие Swagger:

http://127.0.0.1:8000/docs

## Важные отличия от Windows
- Сканерные PowerShell-скрипты требуют отдельной замены (WIA только для Windows).
- Пути и регистр символов в Linux чувствительны: использовать точные имена файлов.
- Для shell-скриптов после clone при необходимости выполнить:

```bash
chmod +x scripts/run_api_clean.sh scripts/setup_linux_mint_22_3.sh
```

## Быстрая проверка после запуска
1. Открывается /health endpoint.
2. Доступен /docs.
3. Создаются/читаются файлы в docflow/objects.
4. Отрабатывают API arm/checklist и arm/assist.
